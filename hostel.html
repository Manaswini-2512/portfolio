🏠 Hostel Room Allocation System

A desktop application that automates the end-to-end process of hostel room allocation — replacing manual paperwork with a clean, GUI-driven system built in Java Swing.

TypeDesktop Application
RoleSolo Developer
StackJava · Swing

Overview

Managing hostel room allocations manually is slow and error-prone. This system was built to automate student registration, room assignment, and occupancy tracking through an intuitive desktop GUI.

Built entirely in Java using the Swing framework, the application uses Object-Oriented Programming principles to keep the codebase modular — with clean separation between the UI layer, business logic, and data management.

My Role & Contribution

🖼️ GUI Design

  • Designed all Swing forms for student registration and room management
  • Built interactive tables to display room occupancy status in real time
  • Implemented form validation to prevent invalid data entry
  • Created a clean navigation flow between screens

🧠 Business Logic

  • Designed the allocation algorithm to assign rooms based on availability
  • Modelled Student, Room, and Allocation as distinct OOP classes
  • Handled edge cases: room full, student already allocated, invalid ID
  • Implemented search and filter functions for admin use

Key Features

  • Student registration with profile data management
  • Automated room assignment based on real-time availability
  • Occupancy tracking dashboard for administrators
  • Search and filter students by name, room, or status
  • Modular OOP architecture — easy to extend and maintain

Challenges & How I Solved Them

🔄 Keeping UI in Sync

  • After any allocation change, multiple UI panels needed to update simultaneously
  • Implemented an observer-style refresh pattern — any data change triggers a UI reload across all relevant panels

🏗️ OOP Class Design

  • Deciding how to structure Student, Room, and Hostel classes without tight coupling
  • Used composition and clear interfaces so each class had a single responsibility
