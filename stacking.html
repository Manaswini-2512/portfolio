🎮 Stacking Plates

A strategic, multi-level puzzle game where players stack plates under time pressure — complete with a leaderboard, undo moves, and progressively harder levels.

TypeDesktop Game
RoleSolo Developer
StackPython · Pygame

Overview

Stacking Plates is a logic-based puzzle game inspired by classic tower-stacking mechanics. Players must arrange plates in the correct order across multiple levels, each with increasing complexity and tighter time limits.

The game was built entirely in Python using Pygame, focusing on clean game loop architecture, real-time rendering, and a satisfying player experience with features like undo and persistent leaderboards.

My Role & Contribution

🎮 Game Design

  • Designed the core puzzle mechanics and level progression
  • Defined win/loss conditions and difficulty scaling across levels
  • Created the undo system using a stack data structure
  • Balanced timer values to keep gameplay challenging but fair

💻 Engineering

  • Implemented the full game loop: input handling, update, render
  • Built the leaderboard with file I/O for persistent high score storage
  • Managed game states — menu, playing, paused, game over
  • Rendered all game elements using Pygame's drawing and sprite APIs

Key Features

  • Multiple levels with progressively increasing difficulty
  • Countdown timer that adds urgency to each level
  • Undo functionality powered by a stack data structure
  • Persistent leaderboard that saves top scores between sessions
  • Clean game state management — menu, playing, paused, game over

Challenges & How I Solved Them

⏱️ Real-Time Game Loop

  • Keeping the timer, animations, and input handling in sync without lag was a challenge
  • Used Pygame's Clock.tick() to cap frame rate and decouple game speed from hardware

↩️ Undo System Design

  • Needed a reliable way to revert moves without duplicating state everywhere
  • Implemented a command-history stack — each move pushed a snapshot, undo popped it
